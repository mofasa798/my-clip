# Bootstrap Prompt

Before writing or modifying code:

1. Read INDEX.md.
2. Read all required context documents.
3. Follow the current roadmap phase.
4. Follow architecture decisions.
5. Keep implementations simple.
6. Avoid unnecessary abstractions.
7. Prefer maintainability over flexibility.
8. Never implement future phases.
9. Never introduce dependencies without justification.
10. Ask for clarification if requirements are ambiguous.

The documentation inside `.contexts/` is the single source of truth for this project.
